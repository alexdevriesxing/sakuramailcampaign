# Security Policy

Please report suspected vulnerabilities privately to alexdevriesxing@gmail.com.

Do not open a public GitHub issue containing exploit details, credentials, personal data or live recipient information.

Include:

- The affected route or component.
- Reproduction steps using test data.
- Expected and observed impact.
- Any suggested mitigation.

This project does not promise perfect or “100%” security. Production launch requires independent review, least-privilege Cloudflare and GitHub access, hardware-key MFA, secret rotation, monitoring, backups and an incident-response process.
